# Ticket API Map

# POST /api/events/:eventId/tickets

## Route Description

* This route handles the creation of a new ticket assigned to the event identified by `:eventId`.
* Staff/admin view.

## Auth Requirements

* Requires `authenticate`.
* Requires `authorize`.
* Permission: `tickets.create`

## Example Request Body

```json
{
  "status_id": 1,
  "ticket_name": "General Admission",
  "ticket_description": "Standard entry ticket.",
  "ticket_price": 25.00,
  "discount_percentage": null,
  "discount_fixed": null,
  "quantity_available": 100,
  "sale_start": "2026-06-28T14:00:00Z",
  "sale_end": "2026-06-29T14:00:00Z",
  "min_per_order": 1,
  "max_per_order": 4,
  "category_ids": [1, 3],
  "allowed_role_ids": [2, 4]
}
```

## Data Written

### TO tickets

* ticket_id — generated by database
* event_id — from `:eventId`
* status_id — defaults to Draft if not provided
* ticket_name
* ticket_description
* ticket_price
* discount_percentage
* discount_fixed
* quantity_available
* quantity_sold — defaults to `0`
* quantity_reserved — defaults to `0`
* sale_start
* sale_end
* min_per_order
* max_per_order
* created_by — current authenticated user
* updated_by — current authenticated user
* created_at
* updated_at

### TO ticket_categories

* ticket_id — generated ticket ID
* ticket_category_id — from `category_ids`

### TO ticket_allows_role

* ticket_id — generated ticket ID
* role_id — from `allowed_role_ids`

## Example Response

```json
{
  "success": true,
  "message": "Ticket created successfully.",
  "data": {
    "event_id": 1,
    "ticket_id": 12
  }
}
```

# PATCH /api/events/:eventId/tickets/:ticketId

## Route Description

* This route handles partial editing of a ticket assigned to the event identified by `:eventId`.
* Only provided fields should be changed.
* Staff/admin view.

## Auth Requirements

* Requires `authenticate`.
* Requires `authorize`.
* Permission: `tickets.edit`

## Example Request Body

```json
{
  "ticket_name": "VIP Admission",
  "ticket_price": 75.00,
  "quantity_available": 50,
  "max_per_order": 2,
  "category_ids": [2],
  "allowed_role_ids": [2, 3]
}
```

## Data Updated

### TO tickets

* status_id
* ticket_name
* ticket_description
* ticket_price
* discount_percentage
* discount_fixed
* quantity_available
* sale_start
* sale_end
* min_per_order
* max_per_order
* updated_by — current authenticated user
* updated_at

### TO ticket_categories

* If `category_ids` is provided, replace the existing ticket category mappings.

### TO ticket_allows_role

* If `allowed_role_ids` is provided, replace the existing ticket role mappings.

## Example Response

```json
{
  "success": true,
  "message": "Ticket updated successfully.",
  "data": {
    "event_id": 1,
    "ticket_id": 12
  }
}
```

# PUT /api/events/:eventId/tickets/:ticketId/categories

## Route Description

* This route replaces the categories assigned to the ticket identified by `:ticketId`.

## Auth Requirements

* Requires `authenticate`.
* Requires `authorize`.
* Permission: `tickets.edit`

## Example Request Body

```json
{
  "category_ids": [1, 2, 4]
}
```

## Data Updated

### TO ticket_categories

* Delete existing mappings for `ticket_id`.
* Insert one row for each provided `ticket_category_id`.

```json
    "event_id": 1,
    "ticket_id": 12,
    "category_ids": [1, 2, 4]

}
```

# GET /events/:eventId/tickets

## Route Description

* This route retrieves public-facing ticket options for the event identified by `:eventId`.
* This route should only return tickets that are published, open, available, and valid for the current user context.
* If a user is logged in, ticket visibility should be filtered using `ticket_allows_role`.
* If a user is not logged in, only unrestricted public tickets should be returned.

## Auth Requirements

* Publicly accessible.
* Authentication is optional.
* Returned tickets depend on the user’s logged-in status and roles.

## Query Parameters

```txt
?quantity=2
```

## Data Retrieved

### FROM tickets

* ticket_id
* event_id
* status_id
* ticket_name
* ticket_description
* ticket_price
* quantity_available
* quantity_sold
* quantity_reserved
* sale_start
* sale_end
* min_per_order
* max_per_order

### FROM ticket_allows_role

* ticket_id
* role_id

## Example Response

```json
{
  "data": [
    {
      "event_id": 1,
      "ticket_id": 12,
      "ticket_name": "General Admission",
      "ticket_description": "Standard entry ticket.",
      "ticket_price": "25.00",
      "remaining_quantity": 75,
      "sale_start": "2026-06-28T14:00:00Z",
      "sale_end": "2026-06-29T14:00:00Z",
      "sales_are_open": true,
      "min_per_order": 1,
      "max_per_order": 4,
      "user_can_purchase": true
    }
  ]
}
```
